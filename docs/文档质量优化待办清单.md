- [x] 交付物 1：软件代码文档（最终可导出 `.docx`）。
- [x] 交付物 2：软件说明文档（最终可导出 `.docx`）。

## 二、软件代码文档优化
- [x] 增加页眉模板渲染逻辑：软件全称、版本号、页码自动注入。
- [x] 增加版权声明一致性检查：若项目有版权声明，校验与著作权人一致。
- [x] 增加代码选择策略：
- [x] 当源代码总行数 `< 3000` 时，输出全部代码。
- [x] 当源代码总行数 `> 3000` 时，输出开头、结尾、主要功能模块。
- [x] 强制最后一页落在“完整模块结尾”。
- [x] 增加版式控制：每页 50 行。
- [x] 增加页数约束：默认目标 60+ 页，硬上限不超过 70 页。
- [x] 增加超限处理：超过上限时按优先级裁剪非核心片段并保留主功能链路。
- [x] 增加“代码文档与说明文档关联标记”：模块名、功能点编号、章节引用。

验收标准：
- 随机抽查 3 页，单页代码行数均为 50 行（允许页尾不足 50 行的最后页）。
- 当输入样本代码 `> 3000` 行时，文档包含开头、结尾、主要模块且最后一页为完整模块结尾。
- 文档总页数 `<= 70`，且核心功能代码均被覆盖。

## 三、软件说明文档优化
- [x] 增加固定章节骨架：登录、主界面、界面跳转、主要功能模块。
- [x] 增加“主要功能全覆盖”校验：每个主要功能至少 1 张截图 + 文字说明。
- [x] 增加截图一致性检查：软件名称、版本号、状态栏/信号栏与申请软件一致。
- [x] 增加截图质量检查：禁止明显 PS 痕迹、明显剪切痕迹。
- [x] 增加安卓/iOS 截图差异说明区块（按审核规则自动生成或半自动填充）。
- [x] 增加页眉模板渲染逻辑：软件全称、版本号、页码自动注入。

验收标准：
- 文档中存在并完整覆盖四类核心章节：登录、主界面、跳转、主要功能。
- 主要功能章节截图齐全，无缺失项。
- 抽查截图能识别系统特征，不出现明显拼接/裁剪痕迹。
