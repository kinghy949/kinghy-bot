## 2. 任务卡（按实现顺序）

### T001 页眉统一能力
- 目标：两类文档页眉统一为 `软件全称 + 版本号 + 页码`。
- 影响文件：
- `server/generators/source_doc_generator.py`
- `server/generators/manual_doc_generator.py`
- `server/generators/models.py`（若缺版本字段则补齐）
- 实现要点：
- 抽取通用页眉注入函数，避免重复实现。
- 页码使用 Word 域（`PAGE`）而不是静态数字。
- 验收标准：
- 任一生成任务的两份 `.docx` 均包含一致页眉字段且版本号一致。

### T002 代码文档行数分支与页数上限控制
- 目标：满足 `<3000` 全量、`>3000` 取开头+结尾+主要功能模块；每页 50 行；页数不超过 70。
- 影响文件：
- `server/generators/source_doc_generator.py`
- `server/generators/code_generator.py`（必要时补充模块元信息）
- 实现要点：
- 增加代码选择器：按“开头段、结尾段、核心模块段”组合输出。
- 核心模块识别支持按路径关键词与功能名匹配双策略。
- 文档渲染时增加分页控制（50 行/页）与超限裁剪策略。
- 验收标准：
- 使用 `<3000` 与 `>3000` 两组样例均通过规则检查，页数 `<=70`。

### T003 说明文档章节骨架与截图覆盖校验
- 目标：固定覆盖登录、主界面、跳转、主要功能模块，且主要功能截图齐全。
- 影响文件：
- `server/generators/manual_doc_generator.py`
- `server/generators/screenshot_service.py`
- 实现要点：
- 输出前检查 feature 列表与截图映射完整性。
- 缺图时触发警告并附“待补截图项”列表。
- 验收标准：
- 说明文档固定章节存在；主要功能均至少 1 张图；缺失项可追踪。

### T004 文档规范检查阶段接入编排器
- 目标：在文档生成后、打包前执行统一规则校验。
- 影响文件：
- `server/generators/orchestrator.py`
- `server/generators/consistency_checker.py`
- 实现要点：
- 扩展现有一致性检查为“文档规范检查”子流程。
- 失败输出具体规则编号与失败原因；支持 warning/error 分级。
- 验收标准：
- 规则不通过时任务状态可见明确失败信息，且不会静默成功。

### T005 文档关联性标记
- 目标：代码文档与说明文档具有关联（模块名、功能点编号、章节引用）。
- 影响文件：
- `server/generators/source_doc_generator.py`
- `server/generators/manual_doc_generator.py`
- 实现要点：
- 为 feature 生成稳定编号（如 `F01`）。
- 源码章节处输出“关联功能点：Fxx”。
- 验收标准：
- 任意主要功能都可在两份文档中双向定位。

### T006 质量报告输出
- 目标：生成 `quality_report.md`，写入规则命中、失败项、建议项。
- 影响文件：
- `server/generators/orchestrator.py`
- `server/generators/consistency_checker.py`
- 实现要点：
- 报告进入 `context.output_files` 并随下载包输出。
- 验收标准：
- 下载包中可见质量报告，内容与任务日志一致。

### T007 失败重试与人工补充建议
- 目标：文档校验失败时自动重试 1 次，并输出补充建议。
- 影响文件：
- `server/generators/orchestrator.py`
- `server/task/task_manager.py`
- 实现要点：
- 仅对可恢复错误重试，避免死循环。
- 失败后输出结构化建议清单。
- 验收标准：
- 可恢复问题能自动修复；不可恢复问题可定位并可人工处理。

### T008 安卓/iOS 截图区分说明增强
- 目标：自动生成“系统区分说明”段落并标注判断依据。
- 影响文件：
- `server/generators/manual_doc_generator.py`
- 实现要点：
- 根据状态栏布局、图标风格、时间位置写入说明文本。
- 验收标准：
- 说明文档存在系统区分段，且与截图一致。

### T009 截图异常痕迹检测优化
- 目标：补充明显裁剪/拼接风险的规则化检查。
- 影响文件：
- `server/generators/consistency_checker.py`
- 实现要点：
- 先以元信息和基础图像规则做轻量检测，避免误判率过高。
- 验收标准：
- 风险截图可被标记为 warning，不阻塞正常流程。
