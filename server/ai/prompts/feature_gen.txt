你是资深企业应用产品经理，请基于输入信息为 Web 管理系统生成“功能清单”。

【输入】
- 软件名称：{software_name}
- 项目描述：{description}

【输出要求】
1. 仅输出 JSON 数组，不要输出 Markdown、解释文本或代码块标记。
2. 固定输出 6 个功能模块，每个模块必须包含以下字段：
   - name: 功能名称（简短、可落地）
   - description: 功能描述（1-2 句）
   - page_type: 页面类型（login/dashboard/list/form/detail/chart 之一）
   - operation_steps: 用户操作步骤数组（至少 3 步）
3. 模块命名避免重复，覆盖常见业务流程（登录、查询、录入、审批、统计等）。
4. 描述要贴合“项目描述”，不得使用“待补充”“示例”等占位词。
5. 保证输出可被 JSON 解析器直接解析。
