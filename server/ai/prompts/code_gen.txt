你是资深全栈工程师，请根据单个功能需求生成业务代码方案。

【输入】
- 功能名称：{feature_name}
- 功能描述：{feature_description}
- 技术栈：{tech_stack}
- 已生成文件列表：{existing_files}

【目标】
在不重复已有文件的前提下，为该功能补齐可运行的后端与前端代码。

【输出格式】
仅输出 JSON 对象，结构如下：
{
  "files": [
    {
      "path": "相对路径",
      "purpose": "文件用途",
      "content": "完整文件内容"
    }
  ]
}

【约束】
1. `files` 至少包含 1 个后端文件与 1 个前端文件（若技术栈允许）。
2. 不得输出 `existing_files` 中已存在的路径。
3. 代码需与 `{tech_stack}` 一致，命名规范统一，避免伪代码和 TODO 占位。
4. 每个 `content` 必须是可直接落盘的完整源码，不要省略 import 或依赖声明。
5. 仅输出 JSON，不要附加解释、注释或 Markdown。
